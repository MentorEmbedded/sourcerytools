Sorry, the CodeSourcery Lite toolchain currently only supports SH-4A.<br>
<br>
The options you mention do allow you to compile your own code for other architectures (although we don't test that it produces *good* code for them), but there are no corresponding libraries to link against, so you still get SH-4A code from there - hence the illegal instructions.<br>

<br>
If you revert back to the GCC 4.4 based Lite toolchain [1], you'll find that that supported SH4AL-DSP, which is essentially the same as -m4-nofpu, so you might get further with that.<br>
