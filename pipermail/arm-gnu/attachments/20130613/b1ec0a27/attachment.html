<br>
There are usually differences in the object file, because it includes<br>
things like compilation date in the debugging information.  But I don't<br>
think you should get differences in the actual generated code.  Are you<br>
sure there is nothing changing, like headers, use of __DATE__ macros,<br>
etc.?  Minor changes in the source code can sometimes lead to apparently<br>
large changes in the object code such as different register choices or<br>
different ordering of code (the behaviour of such code is identical, but<br>
it can look very different).<br>
<br>
If you are looking at the final linked code, then remember that the<br>
order of the object files in the link can often affect the order of the<br>
generated code.<br>
