I have code (linked below) that only runs correctly when compiled 
only with arm-none-eabi version 2009q3-68; the application fails when 
compiled with
<u>both newer and older versions</u> of arm-none-eabi.  <br /> <br />
The problems were observed on TI's AM1707 EVM Rev. H (ARM926EJ-S core). 
 The code is a bare metal application that runs the EVM board as a 
generic USB bulk device, with console output via serial port 
interrupts.  Using the same code and EVM board, different
 behavior is obtained when using different compiler versions (no 
internal compiler errors for any version):<br /> <br /> <u>Older CodeSourcery versions</u> (2009q1-161 and older, through 2008) 
  <ul type="disc">
    <li class="MsoNormal">
Everything works fine when run from debugger, but...</li>
    <li class="MsoNormal">
USB enumeration fails, but only when application is flashed to the board</li>
  </ul> 
  <p><u>Newer CodeSourcery versions</u> (2010q1-188 to 2011.09-69)</p> 
  <ul type="disc">
    <li class="MsoNormal">
Serial port outputs as gibberish via interrupts (both from debugger and when flashed)</li>
    <li class="MsoNormal">
USB enumeration fails (both from debugger and when flashed)</li>
  </ul> 
  <p class="MsoNormal"><u>CodeSourcery G++ Lite version 2009q3-68</u></p> 
  <ul type="disc">
    <li class="MsoNormal">
Everything works correctly! 
      <ul type="circle">
        <li class="MsoNormal">
Serial port output works correctly (both from debugger and when flashed)</li>
        <li class="MsoNormal">
USB enumeration and data transfer work correctly (both from debugger and when flashed)</li>
      </ul> 
    </li>
  </ul> Source code + makefile + long description
