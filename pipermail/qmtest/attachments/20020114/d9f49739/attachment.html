<P><FONT SIZE=2 FACE="Arial">The body of the program test.tcl is:</FONT>

<P><FONT SIZE=2 FACE="Arial">This should simply write out the command line arguments to test.txt and exit with exit status 1.  That is, the test should always fail as far as qmtest is concerned.</FONT></P>

<P><FONT SIZE=2 FACE="Arial">When I run the test, qmtest reports that the test passes.  When I examine test.txt I find only the path to the script (argv0) written out; the command line argument "a" doesn't make it.</FONT></P>

<P><FONT SIZE=2 FACE="Arial">It appears that qmtest ignores the exit status output at program termination, as well as not passing the command-line arguments specified.</FONT></P>
