<br>
Although others here may have more sage advice, I'll offer this: it's
better to think first about what you want to do rather than how to do
it.<br>
<br>
If you're testing a Unix application (in any language), then it might
be natural to select the ShellScriptTest class for your tests - you can
accomplish a lot with a shell script and many Unix people are
comfortable there.  But that also means you're testing in a Unix world,
so running QMTest under Windows is probably a convenience and not a
requirement. <br>
<br>
On the other hand, if you're testing something like a Java library or a
binary application that has both Win32 and Unix ELF versions, then
testing on both platforms is a requirement.  In that case, selecting
the ShellScriptTest isn't such a wise choice but shell scripts are
inherently Unix things (although that doesn't mean you can't make it
work).<br>
<br>
How about selecting one of the Python test classes?  Those will work on
both, provided you've installed Python on your Windows test machines.
 The command tests also should work fine on both platforms if you just
need to feed some input and grab the output.  It's a trade off
sometimes - deciding what support software the test environment
requires.<br>
<br>
When you get comfortable with QMTest, you can go to the next step and
create your own test classes and resources.  This is pretty fun stuff
and not too difficult.   <br>
<br>
But I'm finding myself that it's not always easy to get from the "what"
to the "how".  <br>
<br>
Just to relate, I have to allow Linux, Solaris, and Windows users to
run my tests (which actually control other tests on remote systems), so
Python is a good portable solution.  However, I've just found out that
I can get a big boost by leveraging some Java libraries that have
already been written.   Jython appears to work great for this, but I'm
wracking my brain trying to figure out where to put the stuff - it's
not Python, it's not shell, it's not an executable.  Should I create a
Jython test class?  Should I just wrap the Java calls in a Jython
script and call the wrapper from my tests?  Should I skip the Jython
and just write a Java wrapper?  It's a tough call and I'll have to
experiment to work it out.<br>
<br>
Anyway, I hope this helps somewhat.  Play around, ask questions, and
share what you learn!  :)<br>
