<font size="-1">I have a large suite of tests that I'd like to run
twice, with my device-under-test configured a different way each time.<br>
<br>
Seems there are three ways to set this up in QMTest:<br>
<br>
1. Use a resource for each device configuration.<br>
<br>
2. Use a prerequisite test for each device configuration.<br>
<br>
3. Copy all the tests and embed the server config code in each.<br>
<br>
Options 1 and 2 require that the tests in each suite be dependent on a
different thing (either a resource or a prereq).  Doesn't that imply
duplication of all tests?<br>
<br>
Would it be better to define a "suite resource": a resource that
applies to all tests in a suite but doesn't require me to go in an
change the resource for every test?<br>
<br>
Is there a better way to approach this?  Some creative use of resources
and context, maybe?  The main goal is to keep my test definitions
exactly as they are and to be able to easily say, "Run that suite
again, but do this first.  Undo the change at the end of the suite." 
To me, that sounds like a "suite resource" but I don't see that concept
in the documentation.<br>
