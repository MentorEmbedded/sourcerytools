<font size="-1">Wondering if anybody would be interested in annotated
expectations.  The way I see it working is that a text box would be
added to the "Set Expectation" dialog box.  The text would be saved in
the expectations file (?).  This would allow the user to comment on the
reason for the expected outcome. <br>
<br>
I'd use this, say, to enter some background on why one of my tests is
expected to have an "Error" outcome.  The test is broken but I can't
fix it right now and I'd like to comment on the reason for the
breakage.   Later (weeks/months/years), I can run the tests, load the
expectations, see that everything is normal, and then click on the
broken test's expectation to find out what the heck I was thinking when
I decided an Error was OK.<br>
<br>
Thoughts?<br>
