<BR><FONT SIZE=2> * The types of domains can be divided into two classes, namely integer</FONT>

<BR><FONT SIZE=2> * - Range<Dim></FONT>

<BR><FONT SIZE=2> * domain classes, namely NullDomain, AllDomain and ErrorDomain.</FONT>
<BR><FONT SIZE=2> *</FONT>
<BR><FONT SIZE=2> * All domain classes come with their corresponding traits class</FONT>

<BR><FONT SIZE=2> *   using the equivSubset function</FONT>
<BR><FONT SIZE=2> * - remove the overlap between two Interval<Dim> and return a vector of the</FONT>
<BR><FONT SIZE=2> *   resulting domain using the DomainRemoveOverlap function</FONT>
<BR><FONT SIZE=2> * - querying if one domain contains another using the contains function</FONT>
<BR><FONT SIZE=2> *</FONT>

<P><FONT SIZE=2>/**</FONT>
<BR><FONT SIZE=2> * @defgroup Layout Layouts - Laying out Domains</FONT>
<BR><FONT SIZE=2> * @ingroup Structuring</FONT>
<BR><FONT SIZE=2> *</FONT>
<BR><FONT SIZE=2> * A layout combines domain information and guard cell (both internal</FONT>
<BR><FONT SIZE=2> * and external) information and maps the domain to a specified partition.</FONT>
<BR><FONT SIZE=2> * So related topics are the \ref Domain and \ref Partition groups.</FONT>
<BR><FONT SIZE=2> *</FONT>
<BR><FONT SIZE=2> * All layouts operate with domains based on Interval<Dim>, i.e. a</FONT>
<BR><FONT SIZE=2> * continuous integer domain. This domain gets distributed/tiled in a</FONT>
<BR><FONT SIZE=2> * different way for different layout classes, namely</FONT>
<BR><FONT SIZE=2> * - DomainLayout does not tile/distribute the domain, hence it is applicable</FONT>
<BR><FONT SIZE=2> *   for local computation only</FONT>
<BR><FONT SIZE=2> * - GridLayout tiles/distributes the Dim-dimensional domain by constructing</FONT>
<BR><FONT SIZE=2> *   a (possibly) non-uniform rectilinear grid of subdomains (GridTag)</FONT>
<BR><FONT SIZE=2> * - UniformGridLayout tiles/distributes the domain using a uniform rectilinear</FONT>
<BR><FONT SIZE=2> *   grid of subdomains (UniformTag)</FONT>
<BR><FONT SIZE=2> * - SparseTileLayout tiles/distributes the domain using non-overlapping</FONT>
<BR><FONT SIZE=2> *   subdomains that need not cover the whole domain (SparseTileTag)</FONT>
<BR><FONT SIZE=2> * - DynamicLayout tiles/distributes the domain using a grid partition and</FONT>
<BR><FONT SIZE=2> *   handles dynamic domains such as coming from DynamicArray (DynamicTag)</FONT>
<BR><FONT SIZE=2> *</FONT>
<BR><FONT SIZE=2> * For actually using any of the above layout on a MultiPatch engine you need</FONT>
<BR><FONT SIZE=2> * to specify appropriate grid tags which are one of those specified above</FONT>
<BR><FONT SIZE=2> * alongst the layout classes.</FONT>
<BR><FONT SIZE=2> *</FONT>
<BR><FONT SIZE=2> * Specifying the actual tiling is done by several constructors dealing with</FONT>
<BR><FONT SIZE=2> * the most important tiling types. The generic way to specify tiling is to</FONT>
<BR><FONT SIZE=2> * use a partitioner, see \ref Partition for reference.</FONT>
<BR><FONT SIZE=2> *</FONT>
<BR><FONT SIZE=2> * Mapping the domain to the tiling can be done in two different ways,</FONT>
<BR><FONT SIZE=2> * namely distributed and replicated which is specified using an instance</FONT>
<BR><FONT SIZE=2> * of the DistributedTag and ReplicatedTag classes to the layout constructors.</FONT>
<BR><FONT SIZE=2> * Distributed means the domain is distributed over the tiling, replicated</FONT>
<BR><FONT SIZE=2> * means the entire domain is replicated over the tiling. Note that remote</FONT>
