<BR><FONT SIZE=2>>I tried to use MPI to parallelize a pooma application and stumbeled</FONT>
<BR><FONT SIZE=2>>over the following problem:</FONT>
<BR><FONT SIZE=2>></FONT>
<BR><FONT SIZE=2>>A simple mpirun -np 4 myprog -mpi does not work, the first process</FONT>
<BR><FONT SIZE=2>>gets spawned, but just runs as in uniprocessor mode. The problem is,</FONT>
<BR><FONT SIZE=2>>the spawned processes dont get the -mpi parameter and such dont</FONT>
<BR><FONT SIZE=2>>call MPI_Init.</FONT>
<BR><FONT SIZE=2>></FONT>
<BR><FONT SIZE=2>>The problem is in Cheetah ControllerFactory::create which _removes_</FONT>
<BR><FONT SIZE=2>>the controller argument (-mpi) from the argument list and as such</FONT>
<BR><FONT SIZE=2>>the master MPI_Init and any further MPI_Init's dont see it.</FONT>
<BR><FONT SIZE=2>></FONT>
<BR><FONT SIZE=2>>I worked around this by manually re-inserting -mpi to the arguments</FONT>
<BR><FONT SIZE=2>>list in my program, but of course this is not a solution to the problem.</FONT>
<BR><FONT SIZE=2>>A fix would be to move the argument removal in Cheetah to _after_ the</FONT>
<BR><FONT SIZE=2>>MPI_Init call.</FONT>
<BR><FONT SIZE=2>></FONT>
<BR><FONT SIZE=2>>Anyone with a different solution to the problem? I'll happily contribute</FONT>
<BR><FONT SIZE=2>>the above suggestion, if nobody has a fix already.</FONT>
