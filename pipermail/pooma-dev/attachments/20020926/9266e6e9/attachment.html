<P><FONT SIZE=2>Within the ide subdirectory of the POOMA source tree is</FONT>
<BR><FONT SIZE=2>a VisualC++ directory with VC++ 6.0 workspace and project</FONT>
<BR><FONT SIZE=2>files for building the Pooma library and the test and</FONT>
<BR><FONT SIZE=2>example codes.  I am not sure that these files were</FONT>
<BR><FONT SIZE=2>completely updated between the Pooma 2.3 and 2.4 releases,</FONT>
<BR><FONT SIZE=2>so some project files may require updating.  However, the</FONT>
<BR><FONT SIZE=2>VC++ 6.0 compiler did not have the necessary compliance</FONT>
<BR><FONT SIZE=2>with the ANSI/ISO C++ standard to compile Pooma code.</FONT>
<BR><FONT SIZE=2>Therefore, we had been using the Intel VTune C++ compiler</FONT>
<BR><FONT SIZE=2>within the VC++ IDE as our compiler.  In the src/arch/Intel</FONT>
<BR><FONT SIZE=2>subdirectory, you will see a PoomaConfiguration.h file</FONT>
<BR><FONT SIZE=2>provided for use with the Intel C++ compiler.  You don't</FONT>
<BR><FONT SIZE=2>need to run a configuration script to produce this file.</FONT>
</P>

<P><FONT SIZE=2>If you have the Intel C++ compiler, you can select this</FONT>
<BR><FONT SIZE=2>compiler within the VC++ IDE, load the VisualC++ workspace</FONT>
<BR><FONT SIZE=2>files for the Pooma library and test codes, and build them</FONT>
<BR><FONT SIZE=2>straightforwardly using the VC++ IDE.  If you don't have</FONT>
<BR><FONT SIZE=2>the Intel C++ compiler, another Windows option is to use</FONT>
<BR><FONT SIZE=2>the CodeWarrior compiler from Metrowerks.  There are ide</FONT>
<BR><FONT SIZE=2>files for this compiler available as well.</FONT>
</P>

<P><FONT SIZE=2>I am currently participating in beta testing of the upcoming</FONT>
<BR><FONT SIZE=2>MS Visual Studio .NET compilers.  The version of VC++ in</FONT>
<BR><FONT SIZE=2>this package appears to have the necessary ANSI/ISO C++</FONT>
<BR><FONT SIZE=2>standard compliance required for Pooma.  I plan to make</FONT>
<BR><FONT SIZE=2>available an updated set of VisualC++ ide files for Pooma 2.4</FONT>
<BR><FONT SIZE=2>and this new compiler when the compiler is released (later</FONT>
<BR><FONT SIZE=2>this year?).  So that is the current Windows status.</FONT>
</P>
