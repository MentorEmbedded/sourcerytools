<P><FONT SIZE=2>I think it is unfortunate that we're using "-shmem" here. SHMEM is Cray's system for making a logical memory space on a set of distributed memory computers. It provides high-performance one-sided messaging. If we ever get back to running on Cray's, it would probably be a good idea to put Cheetah on SHMEM, but I don't think there is a version that does that right now. mm simply implements shared memory between forked processes. </FONT></P>

<P><FONT SIZE=2>Mike thinks that the term "shmem" has been used more generically for one-sided messaging using shared memory, which is probably why Cheetah chose it, but a quick web search turned up hits that deal almost exclusively with Cray's SHMEM, so if we tell folks that POOMA or Cheetah uses shmem it could be quite misleading. </FONT></P>

<P><FONT SIZE=2>(Actually, Burton Smith from Cray was through here the week before last and mentioned that shmem is being implemented on other vendors' hardware and so it might be a good thing to explore at some point, but I don't think we want to go there right now.)</FONT></P>
